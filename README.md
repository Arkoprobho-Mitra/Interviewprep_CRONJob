## What It Does

Every day at your chosen time, this script:

1. **Picks 35 concept topics** from a curated bank of 124 topics across AI/DS, Full Stack, and DSA
2. **Generates 5 coding problems** from 42 LeetCode-style patterns — fresh, original, medium-to-hard
3. **Calls Claude** to generate deep, interview-quality questions (not definitions — trade-off reasoning, internals, failure modes)
4. **Sends a beautiful HTML email** to your inbox with all 40 questions
5. **Saves the digest as an `.html` file** to your Google Drive with an auto-incrementing serial number
6. **Creates a blank Answers doc** alongside it for your written practice

No server. No deployment. No maintenance. Runs entirely in Google's cloud.

## Setup — 3 Minutes

### Prerequisites
- A Google account
- An [Anthropic API key](https://console.anthropic.com/)

### Step 1 — Create the Apps Script project

1. Go to [script.google.com](https://script.google.com) → **New Project**
2. Paste the entire contents of `Code.gs` into the editor
3. Press `Ctrl+S` to save

### Step 2 — Set your script properties

Go to **Project Settings** → **Script Properties** → **Add property**:

| Property | Value |
|---|---|
| `ANTHROPIC_API_KEY` | `sk-ant-...` |
| `RECIPIENT_EMAIL` | `you@gmail.com` |

> ⚠️ **Never hardcode your API key in the script body.** Script Properties are encrypted and not shared with collaborators.

### Step 3 — Configure the manifest

In Project Settings, enable **"Show appsscript.json manifest file in editor"**, then replace its contents with:

```json
{
  "timeZone": "PUT your Timezone here",
  "dependencies": {},
  "exceptionLogging": "STACKDRIVER",
  "runtimeVersion": "V8",
  "oauthScopes": [
    "https://www.googleapis.com/auth/drive",
    "https://www.googleapis.com/auth/documents",
    "https://www.googleapis.com/auth/gmail.send",
    "https://www.googleapis.com/auth/script.triggers",
    "https://www.googleapis.com/auth/script.external_request"
  ]
}
```

> This explicitly declares all required OAuth scopes. Without this, Apps Script may fail to request `auth/drive` and throw permission errors at runtime.

### Step 4 — Set your Drive folder

In `Code.gs`, find `createDigestAndAnswerFiles()` and replace the folder ID:

```javascript
const folderId = "YOUR_GOOGLE_DRIVE_FOLDER_ID_HERE";
```

To get your folder ID: open the folder in Drive → copy the ID from the URL:  
`https://drive.google.com/drive/folders/`**`1KFI******-OIR******************`**

### Step 5 — Run setup

1. Select `setupTriggers` from the function dropdown → click **▶ Run**
2. Grant all permissions when prompted
3. Run `testDigest()` to fire immediately and verify everything works

**Done.** The digest will now arrive automatically every day.

## How Topic Selection Works

Each day's topics are selected using a **Linear Congruential Generator (LCG)** seeded by the current date:

```javascript
seed = year * 10000 + month * 100 + day  // e.g. 20260326
```

This means:
- **Same day = same topics** (reproducible — useful for debugging)
- **Different day = different shuffle** (deterministic variety)
- **No external state** needed — purely mathematical

Four independent shuffles are run with offset seeds (`seed`, `seed+1`, `seed+2`, `seed+3`) to keep the four topic banks from correlating.

## Customization

### Change the delivery time

In `setupTriggers()`, adjust `.atHour()`:

```javascript
ScriptApp.newTrigger("sendQuestions")
  .timeBased()
  .atHour(6)           // ← change to your preferred hour (24h format)
  .nearMinute(0)
  .inTimezone("Asia/Kolkata")   // ← change to your timezone
  .everyDays(1)
  .create();
```

### Change question counts

In `sendQuestions()`:

```javascript
var aiQs  = generateConceptQuestions_(topics.ai,  "AI/Data Science",       15); // ← change count
var fsQs  = generateConceptQuestions_(topics.fs,  "Full Stack Engineering", 15);
var dsaQs = generateConceptQuestions_(topics.dsa, "DSA Mastery",             5);
```

And in `selectTopicsForToday_()`, match the sample sizes:

```javascript
ai:        seededSample_(AI_TOPICS,        15, seed),
fs:        seededSample_(FULLSTACK_TOPICS, 15, seed + 1),
dsa:       seededSample_(DSA_TOPICS,        5, seed + 2),
```

### Add your own topics

Topics follow the format `["Category", "Topic — details"]`. Add entries to any of the four arrays:

```javascript
var AI_TOPICS = [
  ["Your Category", "Your Topic — with detail on what to cover"],
  // ...
];
```

## Troubleshooting

**`You do not have permission to call DriveApp...`**  
→ Make sure your `appsscript.json` has the `auth/drive` scope declared, then revoke and re-authorize at [myaccount.google.com/permissions](https://myaccount.google.com/permissions).

**`ANTHROPIC_API_KEY not set`**  
→ Check Project Settings → Script Properties. The key must be named exactly `ANTHROPIC_API_KEY`.

**`Claude API error 400`**  
→ Verify the model string in `Code.gs` matches a currently available model. Check [Anthropic's model docs](https://docs.anthropic.com/en/docs/about-claude/models).

**Digest arrives but coding problems section is empty**  
→ Claude returned problems in an unexpected format. The script has a fallback — check your Drive `.html` file, which may show the raw text. Open the Apps Script execution log for the raw Claude response.

**Script times out (6-minute limit)**  
→ Apps Script has a 6-minute execution limit. If Claude API calls are slow, you may hit this. Check the execution log for which call timed out and consider splitting into two triggers.

